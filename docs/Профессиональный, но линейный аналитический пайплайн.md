**2. Шаг 3: нормализация недели → YYYYMM — это потеря данных**

`"2024.05"` → `"202405"` — это **месяц**, не неделя.

Если исходный ключ недельный (W202418), а вы схлопываете до месяца — теряете временну́ю точность и рискуете получить ложные совпадения при merge.

Сохраняете дату выгрузки в метаданных — хорошо. Но нет проверки: "эти данные актуальны для нашего периода анализа?"

#### 10.2 Разделение наблюдение / интерпретация / рекомендация

```
┌─────────────────────┬──────────────────────────────────────────────────┐
│ ТИП                 │ ПРИМЕР                                            │
├─────────────────────┼──────────────────────────────────────────────────┤
│ НАБЛЮДЕНИЕ          │ "Spearman(КИР, ТЗ) = 0.71"                       │
│ (факт из данных)    │ "В категории X КИР% = 8.1% vs. 3.2% в среднем"   │
├─────────────────────┼──────────────────────────────────────────────────┤
│ ИНТЕРПРЕТАЦИЯ       │ "Вероятно, высокий ТЗ → увеличивает списания"     │
│ (гипотеза аналитика)│ ← помечаем как гипотезу, не как факт             │
├─────────────────────┼──────────────────────────────────────────────────┤
│ РЕКОМЕНДАЦИЯ        │ "Пересмотреть нормативы закупок в категории X"    │
│ (действие)          │ "Провести аудит причин в TS-123 до конца квартала"│
└─────────────────────┴──────────────────────────────────────────────────┘
```

#### 10.3 Ограничения анализа

```
Обязательный раздел в любом отчёте:

• Данных о причинах списаний нет → причинно-следственные выводы — гипотезы
• Анализ за период X–Y → выводы могут не распространяться на другие периоды
• Match rate 87% → 13% строк КИР без данных Poteri → возможно смещение
• Выбросы оставлены → средние значения смещены вверх в аномальных регионах
```
